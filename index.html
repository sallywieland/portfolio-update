How I work

The thinking happens before Figma opens.

I'm the only designer at a 15-person agency, so the process has to earn its time. I don't do wireframes. Not to skip steps, but because high-fidelity is the fast iteration now. The figuring-out happens first.

I use Claude to interrogate PRDs: surface the edge cases, find the gaps, and generate realistic content so the layout gets tested before anyone's in a review. A name that's 4 characters vs. 40. A status label that wraps. A dollar amount with 6 digits. That's where dense interfaces fall apart. I'd rather find it in Figma than in QA.

I started at Crafted as a developer, back when everyone and their cousin was doing a coding bootcamp. Lucky timing. That foundation stuck around, and the gap between design and code keeps shrinking: React prototypes, design tokens, and time spent in Cursor and Claude Code when the work calls for it.

Step 01

Interrogate the PRD

I hand Claude the actual PRD and check my read against it: ambiguities, edge cases, the questions the document quietly skips. Before a pixel gets placed.

Step 02

Generate real content

No lorem ipsum. Real-looking content with real-world variance (long names, short names, the ugly edge values) so the layout gets tested by the design instead of by QA.

Step 03

High-fidelity, directly

Straight to high-fidelity in Figma. Wireframes earned their keep when hi-fi was slow. It isn't anymore, and enterprise stakeholders give better feedback on something that looks real anyway.

Step 04

Own it through handoff

Design tokens, component docs, React prototypes where they help. I've been the only designer on every project, so handoff is airtight or it ships wrong. Those are the options.
